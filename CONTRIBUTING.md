# How to Contribute to MrLavaLoba2

First of all, thank you for your interest in contributing to MrLavaLoba2.

Contributions are welcome, including:

* bug reports
* documentation improvements
* example cases
* code improvements in the Python or Fortran implementations
* testing and reproducibility checks

## Reporting bugs

Before opening a new issue, please check whether the problem has already been
reported in the GitHub Issues section of the repository.

When reporting a bug, please include as much relevant information as possible,
for example:

* a clear and descriptive title
* the version or commit used
* the operating system and environment
* whether you are using the Python or Fortran version
* the input files or a minimal example needed to reproduce the problem
* the full error message or unexpected behavior
* screenshots or output files, if useful

If possible, provide a minimal reproducible example.

## Suggesting enhancements

Enhancement suggestions are also welcome. When proposing a new feature or a
substantial change, please describe:

* the problem you want to solve
* the proposed solution
* why it would be useful for the project or its users

## Contributing code

If you want to contribute code:

1. Fork the repository
2. Create a new branch for your changes
3. Make your modifications with clear and focused commits
4. Test your changes as thoroughly as possible
5. Open a pull request

Please ensure that your pull request description clearly explains:

* what problem is being addressed
* what was changed
* whether the change affects the Python version, the Fortran version, or both
* whether example cases or documentation were updated accordingly

## Coding style

When possible:

* keep changes small and focused
* preserve readability and consistency with the existing codebase
* update comments and documentation when behavior changes
* keep the Python and Fortran implementations consistent when they are intended
  to reproduce the same logic

## Documentation

Documentation improvements are very valuable. This includes updates to:

* `README.md`
* `guide.md`
* example cases
* inline code comments and docstrings

## Questions

If you are unsure whether a change is appropriate, feel free to open an issue
to discuss it before preparing a pull request.

Thank you for helping improve MrLavaLoba2.
